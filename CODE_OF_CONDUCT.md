# Code Of Conduct

CYPHES is an educational and open-source project for building agentic coding systems with visible contracts, approvals, and receipts.

We expect contributors to be direct, respectful, and careful with one another's time.

## Expected Behavior

- Be constructive and specific.
- Assume good intent, but verify technical claims.
- Keep disagreement focused on code, design, security, and project direction.
- Credit upstream work and external ideas.
- Respect boundaries around private code, credentials, and proprietary material.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Publishing private information.
- Posting secrets, tokens, or private prompts.
- Introducing copied proprietary source or leaked code.
- Misrepresenting licensing, provenance, or authorship.
- Repeatedly derailing technical discussions after maintainers redirect.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or PRs; restrict participation; or block users when needed to protect the project and contributors.

To report a conduct concern, contact the maintainers privately through GitHub until a dedicated CYPHES contact is published.
